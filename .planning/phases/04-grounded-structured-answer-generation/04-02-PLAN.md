<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Retained evidence text -> model prompt | Retrieved text is untrusted and may contain prompt-injection or irrelevant instructions |
| Model JSON -> validated answer draft | Parsed model output remains untrusted until every citation is checked against the evidence pack |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | E | `skill/synthesis/prompt.py` | mitigate | Serialize evidence as inert quoted context and require JSON-only output so evidence text cannot change tool or workflow behavior. |
| T-04-06 | T | `skill/synthesis/citation_check.py` | mitigate | Reject missing, dangling, or quote-mismatched citations before the answer can be treated as grounded. |
| T-04-07 | S | `skill/synthesis/uncertainty.py` | mitigate | Surface clipping, gaps, heuristic academic matches, and citation-validation failures as deterministic uncertainty notes so weak evidence cannot masquerade as certainty. |
</threat_model>
